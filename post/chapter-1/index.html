About the Project

This project started in 2019 when I was trying to practice my web-scraping and Machine learning skills. It was the first personal project that I had been involved in, and it even was a start-up project idea that couldn’t be accomplished for many reasons. I learnt a lot from this and I dedicated a great amount of time to iterate over the ideas. Suddenly, it ended up being a Spatial Data Science project due to the advantage of the scraped data: geographically coordinates.

This article will describe the analysis process of the last version of the ML model (the one with Spatial Features) and the things I learnt while building and using these tools. I’ll focus just only on the Data Science aspect, context and how this project could add value to society.

CONTEXT:

Lima is a messy place, we have many special (and spatial) features around the city, it’s a highly crowded, unstructured and multicultural place. Literature could split Lima in 4 categories, all of that dealing with the amount of income one has to get for living in each place. Also, bureaucracy is one of the main problems in this city, in this context we could talk about the Valuation Process. When someone wants to sell his house it has to hire a professional appraiser which makes a market analysis and the cost could rise to almost 1500 PEN (~400 USD) where the process lasts almost 1-2 weeks if you want a fair valuation. Why not automate this process? There are few tech tools for this in Perú.

Who needs this tool?

Property Owners: Those who want to value their assets in order to make a financial decision. House Buyers: Families and Corporations that want to be sure they will pay a fair price for the property. Investors: That wants to spot opportunities to buy and sell properties in the city.

What’s the main problem this tool solves?

The main advantage of this tool is to quickly evaluate the square meter price for a House in a specific Zone. This reduces the cost and time for regular valuation methods.

What ended up being:

The GitHub repository is here The project is based on data from 2020 where we have 7762 samples at the beginning. After the data cleaning we got almost 4000 samples to work considering the following:

  • We are only working with Lima Metropolitan Area Data.
  • We eliminate duplicates (there is many duplicate announces in the webpage)
  • We eliminate outliers that are not actually houses but country-clubs, hospitals, hotels, etc.
  • We encountered that there are certain districts where the coordinates are not well specified.
